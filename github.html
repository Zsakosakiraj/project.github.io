A Git és GitHub

A Git és a GitHub használatakor felmerülhet Önben, hogy mi a különbség a kettő között. A Git egy elosztott verziókövetési rendszer (DVCS), amellyel több fejlesztő vagy más közreműködő is dolgozhat egy projekten. Módszert biztosít egy vagy több helyi ág használatára, majd azok leküldésére egy távoli adattárba. A Git kezel mindent a GitHubbal kapcsolatban, ami helyileg történik a számítógépen. A Git által biztosított főbb funkciók a következők: A helyi gépen van telepítve és használva Kezeli a verziókövetést Támogatja az elágaztatást A GitHub egy olyan felhőplatform, amely a Gitet használja alapvető technológiaként. Leegyszerűsíti a projekteken való együttműködés folyamatát, és egy webhelyet, parancssori eszközöket és egy átfogó folyamatot biztosít, amelyekkel a fejlesztők és a felhasználók együttműködhetnek. A GitHub a Git szakaszában korábban említett „távoli adattárként” működik. A GitHub által biztosított főbb funkciók a következők:
  • Hibák
  • Beszélgetések
  • Lekéréses kérelmek
  • Értesítések
  • Címkék
  • Műveletek
  • Villa
  • Projektek
